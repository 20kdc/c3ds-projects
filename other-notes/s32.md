# S32 Format Informal Spec

The S32 format appeared in CE 7 beta 2 (maybe beta 1?) for transparency-capable sprites.

The header is _identical_ to the S16 format. Colour type 4 is used. However, the data offset actually points to a PNG file.

No file length is given, but the CE spec (I kind of forgot to read it and then wrote this instead. Oops!) suggests you're supposed to use the next frame's offset as the end.

The current `s16.py` implementation uses IEND searching instead, but this can be changed if it ends up meaningfully mattering. (It really _shouldn't._)

As it is, `s16tool.py` and the library structure is poorly prepared to handle the changes in format anyway; support has mainly been added for export purposes i.e. the Blender addon.

```
typedef struct {
	uint32_t colourType; // = 4 for S32
	uint16_t frameCount;
	s32_frame_t frames[frameCount];
} s32_header_t;

typedef struct {
	uint32_t pngOffset; // From start of file
	uint16_t width;
	uint16_t height;
} s32_frame_t;
```

The S32 BLK format is the same basic idea with BLK.

```
typedef struct {
	uint32_t colourType; // = 4 for S32
	uint16_t widthBlocks; // 128x128 pixel blocks, you know the drill
	uint16_t heightBlocks;
	uint16_t frameCount;
	s32_frame_t frames[frameCount];
} blk_header_t;
```
